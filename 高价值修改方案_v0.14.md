## 🔴 V2：情景价格锚定至 PE 倍数

**问题**：当前 5-scenario 的 base/bull/bear 价位来自技术面推演（支撑阻力），缺乏基本面支撑。

**改动**：修改 `_build_scenario_narrative()` 中的价格计算逻辑：

```python
# 当前：技术面推演
base_price = price * (1 + ml_7d/100)  # 基于 ML 预测

# 改为：PE 倍数锚定（有 forward_eps 时优先）
fwd_eps = ctx.get("forward_eps", 0)
if fwd_eps > 0:
    bear_price  = fwd_eps * 18   # 衰退倍数
    base_price  = fwd_eps * 24   # 历史均值倍数
    bull_price  = fwd_eps * 30   # 扩张倍数
    extr_price  = fwd_eps * 36   # 泡沫倍数
else:
    # fallback 保留技术面推演
    ...
```

**代码量**：~40 行修改 | **预计耗时**：30 分钟

## 🟡 N1：叙事"So What"推理链升级

**问题**：04-02 报告 CH4 段落 5 段全部是信号罗列（"P/C=0.8，IV=42.7%，IV Rank 28%"），缺少"这意味着什么→应该怎么做"的推理链。

**示例对比**：

当前输出：
> IV 42.7% vs HV30 37.6%，IV-RV 价差 +5.1%（fair）——期权相对历史波动率显著偏贵。

应改为：
> IV 42.7% vs HV30 37.6%，IV-RV 价差 +5.1%——期权定价隐含的恐慌程度超出实际波动 14%。**这意味着**：做市商正在为即将到来的催化剂（见 CH3 FOMC 4/17）预留风险缓冲。**对交易者**：当前环境下卖方策略（short strangle）的期望值为正，但需等 FOMC 落地后 IV Crush 窗口才是最佳入场点。

**改动**：在 `_build_options_narrative()` 和 `_build_macro_narrative()` 每个段落末尾追加 1-2 句推理结论。使用模板：

```python
# "So What" 推理模板
so_what_iv_rv = ""
if iv_rv > 5:
    so_what_iv_rv = (
        f"<strong>交易含义：</strong>期权隐含恐慌超出实际波动 "
        f"{iv_rv/rv*100:.0f}%，卖方策略期望值为正；"
        f"{'但需等' + next_catalyst + '落地后 IV Crush 窗口入场' if next_catalyst else '可考虑 iron condor 捕捉 theta 衰减'}。"
    )
```

**代码量**：~80 行增强（分散在 3 个 _build 函数中） | **预计耗时**：1.5 小时

## 🟡 N2：CH1 核心论点提炼（Top-3 Thesis）

**问题**：7 只蜂各说各的，读者需要自己综合判断。Executive Summary 已有但缺少 "Top 3 核心论点" 提炼。

**改动**：在 `_build_executive_summary()` 中新增逻辑：

```python
# 从 7 只蜂中提取 top-3 最强信号
thesis_candidates = [
    (ctx["oracle"].get("score",0), "期权", oracle_thesis),
    (ctx["scout"].get("score",0), "基本面", scout_thesis),
    (ctx["chronos"].get("score",0), "催化剂", chronos_thesis),
    (ctx["rival"].get("score",0), "量化", rival_thesis),
```

**代码量**：~50 行 | **预计耗时**：45 分钟
